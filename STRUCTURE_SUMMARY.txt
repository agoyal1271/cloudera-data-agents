LAYER 3: HIERARCHY / AGGREGATION DIMENSIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Multi-level aggregation paths (drill-down/roll-up capability):

PLMN (Public Land Mobile Network) — Top Level
│
├─ PLMN-CMM-VM (Basic 3-level hierarchy)
│  ├─ Dimension 1: PLMN ID (network operator)
│  ├─ Dimension 2: CMM ID (Cloud Mobility Manager / compute cluster)
│  └─ Dimension 3: VM ID (Virtual Machine / instance)
│
├─ PLMN-CMU-CMM-VM (With CPPS Management Unit)
│  ├─ Dimension 1: PLMN ID
│  ├─ Dimension 2: CMU ID (CPPS Management Unit)
│  ├─ Dimension 3: CMM ID
│  └─ Dimension 4: VM ID
│
├─ PLMN-CMMA-CMM-VM (With CPPS Measurement Agent)
│  ├─ Dimension 1: PLMN ID
│  ├─ Dimension 2: CMMA ID (CPPS Measurement Agent)
│  ├─ Dimension 3: CMM ID
│  └─ Dimension 4: VM ID
│
├─ PLMN-CMM-VM-HSS (With Home Subscriber Server)
│  ├─ Dimensions 1-3: As above
│  └─ Dimension 4: HSS ID (subscriber database backend)
│     Use case: Which HSS is bottleneck?
│
├─ PLMN-CMM-VM-CPLMN (With Served PLMN)
│  ├─ Dimensions 1-3: As above
│  └─ Dimension 4: CPLMN ID (Visited/Served PLMN)
│     Use case: Roaming analysis per operator
│
├─ PLMN-CMM-VM-QCI (With Quality Class Indicator)
│  ├─ Dimensions 1-3: As above
│  └─ Dimension 4: QCI (Quality of Service class 1-9)
│     Use case: Premium vs. standard service tracking
│
├─ PLMN-CMM-VM-RESLCT (With Reselection Attempt #)
│  ├─ Dimensions 1-3: As above
│  └─ Dimension 4: Reselection Attempt Number
│     Use case: Gateway failover success rates
│
└─ PLMN-CMM-VM-QCI (and more exotic combinations)


AGGREGATION MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Raw Events (Millions per hour)
        ↓
    Counters (sum, sum) — per VM, per 15 min
        ↓ Aggregation: sum
    Per-CMM (sum, avg) — across VMs
        ↓ Aggregation: sum
    Per-PLMN (sum, avg) — across CMMs
        ↓ Drill-down dimensions
    Per-HSS, Per-QCI, Per-CPLMN, Per-Reselection Attempt
        ↓
    Measurements (KPI formulas)
        ↓
    Storage (Iceberg → Snowflake/Databricks)


COUNTER → MEASUREMENT RELATIONSHIP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Example: M103 (EPS CPPS Bearer Session Management)

Counter Inputs:
├─ ATT_CREATE_DFLT_BRR         (Default bearer creation attempts)
├─ ATT_DEL_DFLT_BRR            (Default bearer deletion attempts)
├─ ATT_BRR_RES_MOD_REQ         (Bearer resource modification requests)
├─ ATT_MOD_ACC_BRR_REQ         (Access bearer modification requests)
├─ ATT_MOD_BRR_REQ             (Bearer modification requests)
└─ ATT_UPD_BRR_REQ             (Bearer update requests)

Aggregation Logic:
  M103 = sum(all 6 counters) per 15-min interval

Busy Hour Definition:
  Busy Hour = max(ATT_CREATE_DFLT_BRR + ATT_DEL_DFLT_BRR + ...)

Output:
  └─ M103 Value: 45,230 bearer operations in a 15-min window


SUCCESS/FAILURE COUNTER PATTERNS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

For each major procedure, counters follow:

  SUCC_<PROCEDURE>
  FAIL_<PROCEDURE>_DUE_<REASON>

Examples:
  ├─ SUCC_GPRS_ATTACH / FAIL_GPRS_ATTACH_DUE_MS_ERR
  ├─ SUCC_GPRS_ATTACH / FAIL_GPRS_ATTACH_DUE_RADIO_ERR
  ├─ SUCC_GPRS_ATTACH / FAIL_GPRS_ATTACH_DUE_SGSN_ERR
  ├─ SUCC_GPRS_ATTACH / FAIL_GPRS_ATTACH_HLR_VLR_ERR
  ├─ SUCC_GPRS_ATTACH / FAIL_GPRS_ATTACH_COLLISIONS
  │
  ├─ SUCC_COMBINED_ATTACH / FAIL_COMBINED_ATTACH
  ├─ SUCC_IMSI_ATTACH / FAIL_IMSI_ATTACH
  │
  ├─ SUCC_INTRA_PAPS_RA_UPDATE / FAIL_INTRA_PAPS_RA_UPDATE
  ├─ SUCC_INTER_PAPS_RA_UPDATE / FAIL_INTER_PAPS_RA_UPDATE
  └─ ... (many more)

Use case: Calculate success rate = SUCC / (SUCC + FAIL) per dimension

DATA INTEGRATION PATTERNS
================================================================================

Ingestion Pipeline (Proposed):

  Mavenir RCS Files (Kafka → 10M events/sec)
              ↓
  Bronze Layer (Flink/NiFi processing)
              ↓
  Raw Counters in Iceberg (7,746 counter types)
              ↓
  Silver Layer (Aggregation by hierarchy)
  - Normalize dimensions (PLMN, CMM, VM, HSS, QCI)
  - Apply time aggregation (15-min windows)
  - Apply object aggregation (sum/avg/max)
              ↓
  Iceberg Silver Tables
  - rcs_counters_raw (PLMN-CMM-VM-HSS-QCI)
  - rcs_measurements (M101-M123 KPIs)
              ↓
  Gold Layer (KPI Calculations & Alerts)
  - Daily rollups
  - SLA comparisons
  - Anomaly flags
              ↓
  Snowflake + Databricks (BI/ML)
  - BI dashboards (TCO tracking, SLA dashboards)
  - ML models (capacity forecasting, anomaly detection)
              ↓
  Autonomous Agents (MCP-based)
  - SLA Monitor Agent
  - Anomaly Detector Agent
  - Capacity Planner Agent
